## Инструменты (`tools/`)

## Quick rules

- Перед ручной реализацией проверь, нет ли готового инструмента в `tools/`.
- Для токенов/глобальных стилей используй существующий генератор, не делай параллельный пайплайн.
- Если функционала не хватает — расширяй текущий генератор или настройки Vite для SCSS, а не создавай новый обходной путь без необходимости.

## Как использовать AI

- Если задача про дизайн-токены/глобальные стили — начни с `tools/generators/generate-global-scss.ts`.
- Если задача про **использование токенов в `*.module.scss`** — см. [styles.md](./styles.md): явный `@use '@styles' as *;` в начале файла. При необходимости общего препроцессинга для всех SCSS (например `additionalData`) правки **`vite.config.ts`** допустимы и не считаются «нарушением scope» для задачи про стили (см. [rules.md](./rules.md)); в текущем репозитории глобальный инжект SCSS **не настроен**.

Перед ручным кодом **проверить** содержимое `tools/`. Сейчас в репозитории:

| Инструмент                                 | Назначение                                                                                                                                      |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `tools/generators/generate-global-scss.ts` | Генерация **`src/shared/styles/variables.scss`** (SCSS-переменные) и **`src/app/styles/global.scss`** (сброс, CSS variables для тем) из `src/shared/lib/constants/design-tokens.ts` |
| [`vite.config.ts`](../../vite.config.ts) (корень) | Алиасы `@`, `@shared`, `@api`, `@styles` и др. для разрешения путей в импортах (в т.ч. `@use` в SCSS через Vite); опционально — `css.preprocessorOptions.scss`, если понадобится единый префикс для всех `.scss` |

Скрипт: `pnpm generate:tokens` → запускает генератор SCSS (также вызывается из `dev` / `build`).

### Правила

- Не дублировать логику генерации токенов вручную в других скриптах.
- Не плодить отдельные пайплайны инжекта переменных в обход генератора и Vite без необходимости.
- Если поведения не хватает — **расширять** существующий генератор и/или `vite.config.ts` (`css.preprocessorOptions.scss`), а не создавать параллельный обходной путь.

Примечание: каталог `tools/**` в ESLint игнорируется отдельными правилами; при добавлении кода в `tools` ориентируйтесь на стиль соседних файлов.

## Чек-лист

- [ ] Не добавлен дублирующий генератор/обходной инжект SCSS.
- [ ] Использован существующий инструмент (или расширен он же / Vite-конфиг).
